# 01 - 基础软件全栈技术分析

> 华锐技术栈：ArchforceEuler OS → AMI/AMQ/AMDB → AMI Platform → 业务应用

---

## AMI 分布式低时延消息中间件

![AMI架构](https://archforce-website.oss-cn-hangzhou.aliyuncs.com/AMI/AMI_22.png)

| 指标 | 数值 | 说明 |
|---|---|---|
| 共享内存时延 | **60ns** | 同机进程间，无锁环形缓冲区 |
| 跨主机时延 | **950ns** | 支持RDMA绕过内核 |
| 吞吐量 | **3500万msg/s** | 超高并发 |
| 递交语义 | **Exactly Once** | 不丢不重不乱 |
| RPO/RTO | **0 / <1s** | 零数据丢失，秒级恢复 |
| 可用性 | **99.999%** | 五个9 |
| 部署节点 | **2500+** | 100%安全运行记录 |

**三种通信模式：** 共享内存(60ns) + 可靠组播(微秒级,一对多) + TCP/RDMA(950ns,跨机)

**vs Kafka：** AMI比Kafka快4-5个数量级（纳秒vs毫秒），架构本质不同——Kafka面向大数据吞吐(磁盘顺序写)，AMI面向金融交易时延(内存+零拷贝)。

## AMQ 分布式低时延消息队列

![AMQ架构](https://archforce-website.oss-cn-hangzhou.aliyuncs.com/AMQ/AMQ_22.png)

| 指标 | 数值 |
|---|---|
| 时延 | **30μs**（生产者到消费者） |
| 吞吐 | **>100万msg/s** |
| 主题数 | 支持上万主题 |
| 云原生 | Kubernetes友好，租户管理 |

对标IBM MQ/ActiveMQ/RabbitMQ/RocketMQ/Kafka，在可靠性、可用性、性能方面均有明显提升。支持零数据丢失前提下在线动态弹性扩缩。

**AMI vs AMQ定位：** AMI面向纳秒级关键路径(交易/风控)，AMQ面向微秒级通用场景(异步解耦/流量削峰)。

## AMDB 高性能嵌入式内存数据库

![AMDB架构](https://archforce-website.oss-cn-hangzhou.aliyuncs.com/AMDB/AMDB_08.png)

| 指标 | 数值 |
|---|---|
| CRUD时延 | **0.2~0.8μs**（开启事务） |
| 测试条件 | 单表1000万记录，30字段 |
| 持久化延迟 | 百微秒级（异步到关系型DB） |
| 高可用 | 多副本强一致同步，快照+WAL恢复 |
| 语言 | Java, C++, Python |
| 接口 | ORM, Linq, SQL |

**核心创新：** 嵌入式SDK形式，直接嵌入服务进程内，无锁数据结构+内存分片。解决了独立数据库无法匹配内存计算高性能的问题。

**已用于：** T7/M9/T9的订单中心、资金中心、报盘服务、交易引擎。

1. **AMI是地基** — 60ns共享内存是华锐区别于所有竞品的根本技术壁垒
